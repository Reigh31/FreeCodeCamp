Introduction to Larvel

User authentication is a crucial aspect of web application security. This guide provides a step-by-step approach to implementing user authentication in a Laravel application. By the end of this documentation, you will have a functional authentication system that includes registration, login, and logout features.

Prerequisites

Before starting, ensure you have the following installed:

  • PHP (version 7.3 or higher)
  • Composer
  • Laravel (version 8 or higher)
  • A web server (e.g., XAMPP, WAMP, or Laravel Valet)
  • A database (e.g., MySQL)
Setting up Laravel
  1. Install Laravel

    If you haven't installed Laravel yet, you can do so using Composer

    composer create-project --prefer-dist laravel/laravel your-project-name
  2. Configure Database

    Update your .env file to configure your database connection

    DB_CONNECTION=mysql
    DB_HOST=127.0.0.1
    DB_PORT=3306
    DB_DATABASE=your_database_name
    DB_USERNAME=your_username
    DB_PASSWORD=your_password
  3. Install Laravel UI

    Laravel UI provides a simple way to scaffold all of the routes and views needed for authentication

    composer require laravel/ui
  4. Generate Auth Scaffolding

    You can generate the necessary authentication views using the following command

    php artisan ui vue --auth
  5. Run Migrations

    Run the migrations to create the necessary tables in your database

    php artisan migrate
Confugring Authentication

If you need to customize the authentication logic, you can do so by modifying the LoginController, RegisterController, and ResetPasswordController located in the app/Http/Controllers/Auth directory.

protected function validator(array $data) { return Validator::make($data, [ 'name' => ['required', 'string', 'max:255'], 'email' => ['required', 'string', 'email', 'max:255', 'unique:users'], 'password' => ['required', 'string', 'min:8', 'confirmed'], ]); }
Testing Authentication

Start the Development Server Start your Laravel development server:

php artisan serve
